# 博客使用说明书

> 适用版本:2026-07-18 重构后。配套文档:`功能清单.md`(功能状态与待办)。
> 原则:**内容改动只动 `src/content/` 和少数数据数组,改完 `pnpm build` 验证再推送。**

## 一、日常三板斧

```bash
pnpm dev        # 本地写作预览(热更新)
pnpm build      # 提交前必跑:类型检查 + 构建,0 errors 才算完
pnpm new-post   # 交互式新建文章(自动生成 frontmatter 骨架)
```

推送到 `main` 分支后,GitHub Actions 自动构建并发布到 https://liushenwuzhu-alpaca.github.io/,约 1 分钟。

## 二、写文章

在 `src/content/posts/` 新建 `.md` 文件,frontmatter 字段:

```markdown
---
title: 文章标题          # 必填
published: 2026-07-18    # 必填,决定排序与生长状态
updated: 2026-07-20      # 可选,修订后更新此日期(影响生长状态)
tags: [笔记, Astro]      # 可选,相关文章按共享标签排序
series: 系列名           # 可选,同名系列自动生成系列盒
draft: false             # true 则仅本地可见,不发布
toc: true                # 生成右侧目录
description: 摘要        # 可选,SEO/OG 用
pin: 1                   # 可选,0-99,>0 则置顶到首页「精选文章」,数字小的在前
abbrlink: my-post        # 可选,自定义 URL 别名(小写字母/数字/连字符)
---

正文 markdown。
```

**发布 = 提交推送**。`draft: true` 的文章不会上线。

## 三、各功能怎么用

### 自动运行(零维护)

| 功能 | 触发条件 | 说明 |
|---|---|---|
| 上一篇/下一篇 | 有 ≥2 篇文章 | 单篇时自动隐藏 |
| 相关文章 | 有共享标签的其他文章 | 最多 3 篇 |
| 阅读进度条 | 文章页自动 | 顶部 2px 细线 |
| 生长状态 | 按日期自动 | 🌱 发布<30 天;🌿 90 天内有修订(`updated`);🌳 常青 |
| 修订次数 | 修订提交 >1 次 | 自动链接 GitHub 历史;**需完整 git 历史,不要浅克隆后构建** |
| 节气日期 | 发布日恰逢节气 | 显示 `· 小暑` 等,2000-2099 有效 |
| OG 图 | 每篇文章自动 | 构建时生成 `dist/og/*.png` |
| 书写统计 | 页脚自动 | 「已书写 N 字 · 建站第 N 天」,构建时统计 |
| 朱砂印章 | 页脚常驻 | 无需维护 |
| 随便看看 | 页脚常驻 | 从 atom.xml 随机选文,无需维护 |
| 项目页 | 构建时自动 | 拉取你的 GitHub 公开仓库;失败回退内置列表 |

### 需要喂内容的

**系列文章**:多篇文章 frontmatter 写同一个 `series: 名字`,文章页自动出现编号系列盒。只有 1 篇时不显示。

**此刻/说说** — `src/content/moments/` 下新建 `.md`:

```markdown
---
date: 2026-07-18
---

一句话或几行随想,支持 markdown。
```

`/now/` 按日期倒序自动排列。文件名建议 `YYYY-MM-DD-关键词.md`。

**书单影单** — 编辑 `src/pages/[...lang]/books.astro` 顶部的 `entries` 数组:

```ts
// type: 'book' | 'movie';rating 1-5 可选
const entry = { title: '书名', type: 'book', review: '一句短评', rating: 4 }
```

**装备页** — 编辑 `src/pages/[...lang]/uses.astro` 顶部的 `data` 数组,按分组加条目;空分组不渲染。

**相册** — 把图片丢进 `src/assets/photos/`(支持 jpg/png/webp/avif),重新构建即出现在 `/photos/`。
- 排序:文件名前缀,如 `01-雪山.jpg`、`02-海边.jpg`
- 配文:编辑 `photos.astro` 里的 `descMap`,键为文件名(不含扩展名)

**友链** — 编辑 `src/pages/[...lang]/friends.astro`,按文件内格式加条目。

### 彩蛋

- **隐藏主题**:任意页面按 `↑↑↓↓←→←→BA`,解锁「砂」朱砂夜笺,永久生效(再按不会锁回,主题按钮会多出第四档)
- **暗色问候**:暗色模式下页脚出现「夜深了,写完这篇就睡」;改文案在 `src/components/Footer.astro` 搜 `dark-greeting`

### 评论(Giscus)

文章底部自动加载,访客用 GitHub 账号登录即可评论。管理入口:仓库的 Discussions 页。

### 未启用(需要你的账号)

| 功能 | 启用方法 |
|---|---|
| Umami 统计 | 注册 umami.is(或自托管),建站点拿 website ID,填入 `src/config.ts` 的 `seo.umamiAnalyticsID` |
| Newsletter | 注册 Buttondown/竹白,拿到订阅地址后再接线(当前未做) |

## 四、部署

**GitHub Pages(现状)**:推 `main` 即自动部署,无需操作。

**Docker(上服务器备用)**:

```bash
docker compose up -d --build   # 构建并启动,访问 http://127.0.0.1:8080
```

镜像内是构建好的静态站 + Web 服务器,不依赖宿主机环境。

## 五、维护纪律

1. **推送前 `pnpm build`**,0 errors 再推;`pnpm lint` 保持 0 错误
2. 文章/动态只放 `src/content/`;个人数据(书单/装备/友链)只改对应页面顶部的数据数组
3. 全站只有印章允许硬编码朱砂色 `#B13B2E`,其余配色走 CSS 变量
4. 不要把临时文件、备份文件提交进仓库
5. UI 文案用中文,代码注释用英文
